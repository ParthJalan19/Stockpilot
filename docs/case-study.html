1. Problem Statement

Small to medium enterprises (SMEs) face massive discrepancies in stock management when scaling. Legacy systems like NetSuite or SAP are prohibitively expensive and require months of staff onboarding. Meanwhile, basic tools fail to isolate multi-tenant databases, track multi-warehouse allocations, generate automated QR labels, or log structured before-and-after audit logs. StockPilot was engineered by Nexovian Technologies to solve these issues.

2. SaaS System Architecture

StockPilot leverages a clean, multi-tenant Express REST API routing framework mapped to isolated Mongoose schema structures. Tenancy is strictly enforced across every transaction, listing, search query, and analytics widget by scoping query lookups with the authenticated user's `organizationId` decoded from JWT headers.

3. Database Schema Models

The Mongoose collection layout was structured to handle high-frequency logistics ledger writes while preserving relational integrity:

Collection Name Scoping Type Primary Indexing Elements SaaS Operational Purpose
organizations Root Profile `subdomain` (unique) Stores timezone, currency tags, GST/PAN numbers, tax standard, logo paths.
users Multi-Tenant `email` (unique), `orgId` Stores hashed credentials, session history logs, and RBAC permission flags.
products Multi-Tenant `sku` (unique per tenant), `orgId` Catalog profiles, variant parameters, min/max thresholds, barcodes, manuals.
warehouse_stocks Multi-Tenant `productId` + `warehouseId` (unique) Tracks physical quantity counts and reserved stock allocations at each hub.
inventory_movements Multi-Tenant `productId` + `warehouseId` Immutable ledger logging before/after counts, updatedBy, and operation IDs.

4. Key SaaS Workflows

  • Stock Reservation logic: When a sales invoice is created, items are reserved (`reservedQuantity` increments). This prevents double-selling. Upon payment or completed delivery, the reservation is released, and items are physically deducted from warehouse counts, logging ledger movement logs.
  • Inter-Warehouse transfers: Proposed shipments must specify a source warehouse with verified available stock. Stock is reserved on proposal, deducted on shipment, and added to the destination warehouse upon receiving intake scans.
  • Onboarding Setup Wizard: New signups are guided through localization, establishing default warehouses, default categories, and publishing their first active inventory SKU to prevent blank dashboard displays.

5. Future AI Expansion

StockPilot contains a reserved `backend/ai/` module structure holding forecasting models stubs (`forecast.js`, `reorderSuggestion.js`, `salesPrediction.js`). These hooks are prepared to run linear regression algorithms on historical customer sales and purchase records, alerting managers on reorder dates and warning of prospective supply chain failures.
