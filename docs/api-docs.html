StockPilot REST API

Complete endpoint integration routes and request/response specifications.

Authentication Headers

All API requests (except login and register) must pass the verified user authentication JWT token inside the authorization header field:

Authorization: Bearer <stockpilot_access_token>

Endpoint Specifications

POST /api/auth/register-org

Create a new tenant organization and primary administrator login.

Body Parameters:
`name` (String, required) - Organization name.
`email` (String, required) - Admin user email address.
`password` (String, required) - Password credentials.
POST /api/auth/login

Verify credentials, write access tokens, and record session histories logs.

Body Parameters:
`email` (String, required), `password` (String, required)
GET /api/products

Load paginated product lists scoped to active tenant. Supports search query and stock level filters.

Query Parameters:
`page` (Number), `limit` (Number), `search` (String), `category` (ObjectId), `stockStatus` (String: LowStock/OutOfStock)
POST /api/products

Create a new item entry. Supports uploading product photos (multipart form).

Body Parameters:
`sku` (String, unique), `name` (String, required), `purchasePrice` (Number), `sellingPrice` (Number), `category` (ObjectId)
PUT /api/transfers/:id/status

Transition inter-warehouse transfer stages. Stock reservation updates physically execute on Completed transitions.

Body Parameters:
`status` (String, required) - Valid status options: `Approved`, `Shipped`, `Received`, `Cancelled`
GET /api/settings/backup

Download all data collections linked to active organization as an encapsulated backup file.
